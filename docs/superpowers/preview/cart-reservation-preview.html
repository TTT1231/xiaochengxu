设计原则

  • 只给有效时间：选择器 start/end 约束 → 所选必合法，无错误提示
  • 无有效时间则隐藏：关店或蛋糕来不及时，选择器不显示，只留原因
  • 普通订单：start = 营业开始 与 当前 取大
  • 含蛋糕：start = 营业开始 与 当前+3h 取大
  • end 固定 = 营业结束（22:00）

四种状态

  • 普通：选择器可选 14:00+ → 选完即结算
  • 蛋糕：选择器从 17:00 起（已排除不足3h）
  • 关店前下蛋糕(21:00)：选择器隐藏，红色"蛋糕来不及"
  • 营业已结束(23:00)：选择器隐藏，灰色"今日营业已结束"

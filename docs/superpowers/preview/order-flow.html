下单流程

从用户点击「结算」到订单创建完成。

客户端 API 层 云函数
客户端点击「结算」
校验：购物车非空、用户已登录、submitting 防重复。计算 totalAmount / discountAmount / walletDeduct。
传递 items + 3 个金额 ↓
API精简数据 + 发起调用
CartItem → 只提取 { product_id, specs, quantity }（不发 price / name / image），调用 wx.cloud.callFunction('create-order')
云函数收到请求 ↓
云函数鉴权 + 输入校验
getOpenId() → items 非空且 ≤20 → 三个金额有限、≥0、≤2 位小数
云函数逐项验证商品（事务外·只读）
对每个 item 查 products.doc(id)：存在 + status=true + 价格合法 + 规格合法（未售罄、必填不缺）
客户端仅传 product_id + specs + quantity，name / price / discount / image 全部由服务端从 DB 填充
云函数重算金额 & 比对
expectedTotal = Σ(服务端price × qty)，与客户端传入值比对（误差 > 0.01 则拒绝）。校验 walletDeduct ≤ payableAmount
云函数查找钱包（事务外）
若 walletDeduct > 0：findWalletByUserId(openid) 获取 walletId
TRANSACTION
重读钱包 + 扣款
wallets.doc(id).get() → 验证所有权 + 余额充足 → balance -= walletDeduct
创建订单
orders.add({ _id: orderId, order_status: 'pending', ... }) — 每条商品的 name / price / discount / image 全部由服务端从 DB 填充，客户端无法注入
客户端完成
清空购物车 → 刷新钱包余额 → 跳转订单详情页
安全核心客户端仅发送 product_id + specs + quantity，不传任何价格/名称/图片数据。name、price、discount、image 全部由云函数从 DB 填充到 OrderDetailItem，客户端无法篡改。

金额计算链路

从商品单价到实付金额。

折前总额
totalAmount
−
优惠总额
discountAmount
−
钱包扣款
walletDeduct
=
实付金额
payableAmount
字段 公式 说明
totalAmount Σ(price × quantity) 服务端用 DB 价格重算比对
discountAmount Σ(discount × quantity) 服务端重算比对 (ε=0.01)
walletDeduct min(balance, totalAmount − discountAmount) 上限 = 应付金额
payableAmount max(totalAmount − discountAmount − walletDeduct, 0) 微信支付金额

取消订单 & 退款

用户取消 → 状态变更 + 钱包退款（仅当 wallet_deduct > 0）。

客户端点击「取消订单」
条件 canCancel（pending / preparing）→ 确认弹窗 → 发起请求
API发起取消
callFunction('cancel-order', { orderId })
云函数鉴权 + 预查钱包
getOpenId() → 校验 orderId → findWalletByUserId(openid) 获取 walletId
TRANSACTION
读取订单 + 三重校验
存在 + 归属（user_id === openid，不匹配返回「不存在」防泄露）+ 状态（仅 pending / preparing 可取消）
更新状态 → cancelled
orders.doc(id).update({ order_status: 'cancelled' })
钱包退款
balance += wallet_deduct（total_recharged 不变，它是累计充值额）
客户端刷新页面
重新拉取订单详情 → 显示「已取消」+ 退款金额
原子性保障状态变更 + 退款在同一 runTransaction 中。退款失败则状态不变——要么全部成功，要么全部回滚。

钱包余额操作

wallet.ts 中三个纯函数。total_recharged 只增不减（VIP 累计指标）。

rechargeWallet

balance = + amount
total_recharged = + amount
recharge 云函数

deductWallet

balance = − amount
total_recharged = 不变
create-order（事务内）

refundWallet

balance = + amount
total_recharged = 不变
cancel-order（事务内）
VIP 判定total_recharged > 0 即为 VIP，充值后永不降级。balance 是可用额度，可增可减。

设计权衡

服务端价格验证忽略客户端 price / discount / image，从 DB 重取。防篡改。
原子事务扣款 + 创建订单在 runTransaction 中，保证一致性。
归属校验防越权非本人订单返回「不存在」而非「无权」，防探测。
钱包查找在事务外微信云事务不支持 where()，但事务内重读 + 校验所有权，安全。
商品验证竞态窗口验证和下单间商品可能变化（微信云事务限制），实际影响极小。
无幂等键依赖 submitting flag 防重复，极端情况可创建重复订单。

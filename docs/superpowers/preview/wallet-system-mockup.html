数据流对比
旧积分系统 vs 新钱包系统

旧积分系统

1
新用户登录 → 创建 credits 表 记录
total_scores: 0, available_scores: 0
2
用户下单 → 累积积分
creditsEarned = 订单金额 - 折扣
3
积分 ≥ 阈值 → 升级会员等级
100→黄铜, 200→白银, 300→黄金
4
取消订单 → 扣除积分 + 降级
5
点击余额 → 积分商城（mock数据，不可用）
6
credits 表 独立于 users

新钱包系统

1
新用户登录 → 创建 wallets 表 记录
balance: 0, total_recharged: 0
2
用户充值(模拟) → balance ↑  total_recharged ↑
20/30/50/自定义 → recharge 云函数
3
total_recharged > 0 → 会员
不区分黄铜/白银/黄金
4
取消订单 → 退还余额
wallet.balance += order.wallet_deduct
5
下单时可选 余额抵扣
实际支付 = 总额 - 余额抵扣
6
wallets 表 独立管理余额
数据表变更
表名字段旧新
users _idopenidopenid (不变)
name✓✓ (不变)
level✓ 普通/黄铜/白银/黄金移除
created_at✓✓ (不变)
wallets NEW _id—UUID
user_id—openid (unique)
balance—当前余额 (元)
total_recharged—累计充值 (元)
updated_at—最后更新
orders wallet_deduct—余额抵扣金额 (元)
credits _id✓废弃
users_id✓废弃
total_scores✓废弃
available_scores✓废弃
云函数变更
云函数操作变更内容
recharge NEW 模拟充值，增加 wallet.balance 和 wallet.total_recharged
create-order MOD + walletDeduct 参数 → 扣减余额，移除积分累积/等级升级逻辑
cancel-order MOD 退还余额 (wallet.balance += wallet_deduct)，移除积分扣除/降级逻辑
user-login MOD 创建 wallets 记录替代 credits，用户不再设置 level
get-profile MOD 查询 wallets 替代 credits，返回 wallet 而非 credits
utils/credits.ts DELETED addPoints / subtractPoints / getUpdatedLevel / calculateCreditsEarned
utils/database.ts 清理 移除 LEVEL_THRESHOLDS / getLevelForScore / UserLevel / isValidUserLevel
